# CTF 解題紀錄：Password Is Not Needed (捷克共和國)

## 1. 題目資訊
- **題目名稱**：奪取_捷克共和國 - Password Is Not Needed.
- **分類**：Cryptography (密碼學) / Web
- **分數**：20 分
- **核心漏洞**：PHP Type Juggling (Magic Hash / 弱型別比較)

## 2. 題目分析
### 線索探查
1. **隱藏註解**：在 HTML 原始碼末端發現 Base64 字串，解碼後得到 `/password.php?source`。
2. **後端邏輯**：存取 `?source` 後看到 PHP 原始碼，發現密碼驗證使用 `==`（鬆散比較）：
   ```php
   if ($username == $row["username"] and md5($password) == $row["password"])
   ```
3. **資料庫內容**：其中一位使用者 `sena` 的密碼 Hash 為 `0e959146861158620914280512624073`。

### 漏洞原理
- **PHP 鬆散比較 (`==`)**：當字串符合 `0e` 後面接純數字的格式（科學記號），PHP 會將其視為浮點數處理。
- `0e959146...` 代表 $0 \times 10^{959146...}$，結果為 **0**。
- **攻擊思路**：尋找一個明文字串，其 MD5 運算後的結果也是 `0e` 開頭且後方皆為數字。



## 3. 解題過程
### 錯誤嘗試
- **嘗試輸入**：`username: sena` / `password: 0e830400451993494058024219903391`
- **結果**：失敗。
- **原因**：程式會將輸入的密碼再次進行 `md5()` 運算，導致結果不符合 Magic Hash 特性。

### 正確破解
- **明文字串**：`QNKCDZO`
- **MD5 結果**：`0e830400451993494058024219903391`
- **比較邏輯**：
  `md5("QNKCDZO")` (即 `0e830...`) `==` `0e9591...`
  在 PHP 中等同於：
  `0 == 0` → **True**

## 4. 最終結果
- **登入帳號**：`sena`
- **登入密碼**：`QNKCDZO`
- **Flag**：`flag{Magic_H@sh_l5_vEry_fUn}`

---

## 5. 知識點總結
- **Magic Hash**：在 PHP 弱型別環境下，`0e` 開頭的雜湊值會引發非預期的真值判定。
- **防禦手段**：
  1. 使用 `===` (嚴格比較) 代替 `==`。
  2. 使用 `hash_equals()` 函式進行定時攻擊安全的字串比較。
  3. 使用現代加密演算法如 `password_hash()`。
