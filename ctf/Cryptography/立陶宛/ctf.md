# CTF 解題紀錄：奪取_立陶宛 - Do a robo rotation 2

## 1. 題目資訊
- **題目名稱**：奪取_立陶宛 - Do a robo rotation 2
- **分類**：Cryptography (密碼學)
- **分數**：80 分
- **核心漏洞**：Unicode Rotation (碼位偏移加密) / 8086 偏移量

## 2. 題目分析
### 線索探查
1. **密文特徵**：
   密文內容為 `背胲胧胭脁肻胺胯胒胲胥胙肷胳胶胲肹胥胸胏胭胮胚脃`。
   觀察發現所有字元皆為「肉部」漢字，其 Unicode 碼位集中在 `U+80xx` 至 `U+81xx` 區間。
2. **題目提示**：
   - 「Do a robo rotation 2」：暗示這是一次旋轉（Rotation/Offset）加密。
   - 「Mayhaps more than 8000 times」：在計算機科學中，**8086** 是最具代表性的數字（Intel 8086 架構）。

### 漏洞原理
- **Unicode Caesar Cipher**：
  這類題目將傳統凱撒密碼的範圍由 26 個字母擴展至整個 Unicode 字元集。透過將明文的 ASCII 碼加上一個特定的偏移量（Offset），使其變為漢字或特殊符號。
- **偏移量推導**：
  已知明文開頭應為 `Flag{`。
  - `F` 的 ASCII 碼為 `0x46` (70)。
  - `背` 的 Unicode 碼位為 `0x80CC` (32972)。
  計算兩者差距：$$0x80CC - 0x46 = 0x8086$$
  證實加密邏輯為：$C = P + 0x8086$。

## 3. 解題過程
### 正確破解
- **解碼逻辑**：
  撰寫 Python 腳本遍歷密文字串，將每個字元的碼位減去關鍵偏移量 `0x8086` (十進位 32902)，即可還原回原始 ASCII 字元。

- **解法腳本 (Python)**：
  ```python
  cipher = "背胲胧胭脁肻胺胯胒胲胥胙肷胳胶胲肹胥胸胏胭胮胚脃"
  offset = 0x8086  # 關鍵偏移量

  flag = "".join(chr(ord(c) - offset) for c in cipher)
  print(f"Flag: {flag}")
  ```

## 4. 最終結果
- **關鍵偏移量**：`0x8086` (32902)
- **Flag**：`Flag{5tiLl_S1mpl3_rIghT}`

## 5. 知識點總結
- **Unicode 碼位辨識**：
  當密文呈現整齊的漢字部首分佈時，通常代表碼位偏移量是一個固定的大數。
- **文化與技術暗示**：
  CTF 題目常將解密金鑰或偏移量隱藏在技術名詞中（如 8086, 31337, 404 等），對計算機歷史的敏感度有助於快速定位。
- **防禦手段**：
  Caesar Cipher 及其變體（如 ROT-N）僅具備混淆效果，無法抵禦「已知明文攻擊（KPA）」，不應用於保護任何敏感資料。
