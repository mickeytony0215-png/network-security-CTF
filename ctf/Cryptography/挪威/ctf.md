# CTF 解題紀錄：奪取_挪威 - Nani the fuck ?

## 1. 題目資訊
- **題目名稱**：奪取_挪威 - Nani the fuck ?
- **分類**：Cryptography (密碼學) / Esoteric Language
- **分數**：20 分
- **核心漏洞**：Esoteric Language (奇葩語言) - Brainfuck 混淆

## 2. 題目分析
### 線索探查
1. **特徵觀察**：題目僅給出一串由 `+`, `-`, `<`, `>`, `[`, `]`, `.` 組成的字串。
2. **語言識別**：這是典型的 **Brainfuck** 程式語言。這類語言不具備傳統語法，而是透過操作指標（Pointer）在記憶體單元（Cells）中進行數值增減來實現運算。

### 漏洞原理
- **Brainfuck 混淆**：利用極簡的指令集將明文（Flag）轉換為一系列的記憶體操作。雖然對人類來說難以閱讀，但對於解譯器而言是標準的執行邏輯。其核心原理是將特定數值（ASCII 碼）計算完成後，利用 `.` 指令輸出字元。

## 3. 解題過程
### 正確破解
- **本地解譯腳本**：
  撰寫 Python 腳本實作 Brainfuck 的 8 個標準指令（包含迴圈跳轉表處理），將題目字串作為輸入執行。
- **執行指令**：
  ```bash
  python3 bf_interpreter.py
  ```
- **執行邏輯**：
  1. 初始化 30,000 個數值為 0 的記憶體空間。
  2. 根據 `+` 與 `-` 調整當前空間的數值。
  3. 透過 `[` 與 `]` 進行循環（通常用於快速累加數值以達到字元的 ASCII 範圍）。
  4. 最後經由 `.` 將結果輸出。

## 4. 最終結果
- **執行輸出**：`Flag{mY_bRain_i5_fxCK}`
- **Flag**：`Flag{mY_bRain_i5_fxCK}`

## 5. 知識點總結
- **Esoteric Languages (ESO)**：在 CTF 中，除了 Brainfuck，常見的還有 **Ook!** (由「Ook.」、「Ook?」、「Ook!」組成) 或 **Piet** (以圖片像素組成)。
- **識別技巧**：
  - 看到大量的 `+`、`-` 且伴隨 `[]` -> **Brainfuck**。
  - 看到大量的 `Ook` -> **Ook!** (其實就是 Brainfuck 的變體)。
  - 看到空格、Tab 與換行組成 -> **Whitespace** 語言。
