# CTF 解題思路完整分析：奪取_美國 — Image Downloader 2

**平台**：NSYSU IDSCTF (`idsctf.mis.nsysu.edu.tw`)  
**類別**：Steganography（隱寫術）  
**分數**：80 分  
**Flag 格式**：`Flag{....}`  
**出題者**：苦碩JJ青衫濕

---

## 一、題目資訊與初始觀察

### 題目描述

> *Nope, some pictures have copyright so u cannot download them.*  
> *NOTE: The flag is in the form: Flag{....}*

題目提供一個 `[ Link 1 ]` 連結，指向 `http://idsctf.mis.nsysu.edu.tw:9017`，頁面列出 10 個超連結，每個對應一個「圖片」或影片。

### 初始想法（錯誤的）

> **⛔ 錯誤判斷**：一開始以為「輸入您的答案」是一個「Image Downloader」工具，可以貼入 URL 來下載圖片。因此錯誤地從 SSRF（Server-Side Request Forgery）的角度去思考，想著要繞過 URL 過濾器、用 `file://` 協議讀取本地檔案等。

> **✅ 修正**：後來發現「輸入您的答案」其實是 CTF 的 Flag 提交欄，不是什麼下載工具。「Image Downloader 2」只是題目名稱。正確的做法是先去看 Link 1 裡的圖片列表。

**教訓**：不要過度複雜化題目。先仔細看清楚頁面上每個元素的功能，再開始推理。

## 二、發現異常的思路鏈

### Step 1：逐一檢查 10 個連結

點開每個連結，觀察哪些可以正常下載、哪些不行。

**關鍵發現**：「前鎮區的方俊傑」沒有右鍵另存圖片的選項，其他圖片和影片都可以。

**為什麼這很重要？** 因為題目說「有些圖片有版權所以不能下載」，而無法另存圖片的那個就是「不能下載」的那張。這張圖裡面很可能就藏著 Flag。

### Step 2：分析為什麼不能下載

**推理過程**：

正常圖片是用 `<img>` 標籤顯示的，瀏覽器一定會提供右鍵另存功能。既然不能另存，表示它根本不是真正的圖片檔案。

**驗證方法**：用 `Ctrl+S` 存檔後，發現副檔名是 `.htm`，不是 `.png` 或 `.jpg`。這就確認了——這個「圖片」其實是一個 HTML 網頁。

> **💡 思路關鍵點**：不能右鍵另存 → 不是 `<img>` 標籤 → 是 HTML 網頁 → 檢查原始碼看它怎麼「畫」圖

### Step 3：檢查 HTML 原始碼

**如何取得原始碼**：

- 方法一：在頁面按 `Ctrl+U` 直接看原始碼
- 方法二：用 `Ctrl+S` 存成 `.htm` 檔，再用文字編輯器開啟

**發現的核心結構**：

```html
<div style="height: 1px; width: 1px; box-shadow:
  #000096 0 0,
  #000096 1px 0,
  #000096 2px 0,
  ...
  #fe0000 199px 133px
"></div>
```

**這是什麼？** 這是 CSS Pixel Art 技術。一個 1×1 像素的 `<div>`，透過 `box-shadow` 屬性在每個座標 `(X, Y)` 畫一個指定顏色的像素。總共 200×134 = 26,800 個像素，渲染出一面中華民國國旗（左上藍色 `#000096`，其餘紅色 `#fe0000`）。

**為什麼用這種方式？** 因為 `box-shadow` 不是圖片，瀏覽器不會讓你「另存圖片」，完美呈現題目說的「版權保護不能下載」的效果。

## 三、尋找隱藏資料

### 為什麼知道有隱藏資料？

- 題目類別是 Steganography（隱寫術），代表 Flag 藏在某種媒體裡
- 圖片是用 CSS 逐像素畫的，每個像素都是一個顏色值
- 在這種結構裡，最自然的藏資料方式就是修改某些像素的顏色值

### 怎麼找到異常像素？

國旗的藍色區塊背景色統一為 `#000096`。只要找出不是 `#000096` 的像素，就是被動過手腳的。

**方法一（手動）**：在原始碼裡 `Ctrl+F` 搜尋，排除掉 `#000096`、`#fe0000`、`red` 等正常色碼，找到異常值。但像素數太多，不實際。

**方法二（腳本）**：用 Python 或 JavaScript 遍歷所有像素，自動篩選出異常值。這是 CTF 的標準做法。

### 找到的異常像素

在 `y=2`（第三行）的 `x=50` 至 `x=55` 位置，發現 6 個顏色明顯偏離背景的像素：

| 位置 (x, y) | 正常背景色 | 實際色碼 | 是否異常 |
|---|---|---|---|
| (50, 2) | `#000096` | `#466c61` | ★ 異常 |
| (51, 2) | `#000096` | `#677b33` | ★ 異常 |
| (52, 2) | `#000096` | `#615379` | ★ 異常 |
| (53, 2) | `#000096` | `#5f7231` | ★ 異常 |
| (54, 2) | `#000096` | `#676854` | ★ 異常 |
| (55, 2) | `#000096` | `#3f7d96` | ★ 異常 |

**為什麼肉眼看不出來？** 因為這些異常顏色（如 `#466c61`）和背景色（`#000096`）在視覺上都是深色，混在一起根本分不出差異。而且整張圖只有 200×134 像素，每個像素 1px，非常小。

## 四、解碼過程：RGB → ASCII

### 為什麼知道是 ASCII 字元？

**這不是「知道」，而是「推理 + 嘗試」的過程：**

#### Step 1：分析資料特徵

一個 hex 色碼 `#466c61` 本質上就是 3 個 byte：`0x46`、`0x6c`、`0x61`。在隱寫術中，用顏色通道藏資料最常見的方式：

- **直接當 ASCII**：每個 byte 對應一個字元
- **LSB 隱寫**：只取每個通道的最低位元（Least Significant Bit）

這題異常像素只有 6 個，資料量很少（最多 18 個 byte），不像是 LSB 隱寫。所以先試最直覺的方法：每個 byte 直接當 ASCII。

#### Step 2：試一下就知道了

拿第一個異常色碼 `#466c61` 拆開：

- R = `0x46` = 70，查 ASCII 表 → **「F」**
- G = `0x6c` = 108，查 ASCII 表 → **「l」**
- B = `0x61` = 97，查 ASCII 表 → **「a」**

> **💡 關鍵時刻**：看到 "Fla" 開頭，再對照題目說 Flag 格式是 `Flag{....}`，基本上就確認這個解讀方式是對的！繼續往下解碼就好。

#### Step 3：完整解碼表

| 像素 | Hex 色碼 | R (hex→ASCII) | G (hex→ASCII) | B (hex→ASCII) |
|---|---|---|---|---|
| x=50 | `#466c61` | `0x46` → **F** | `0x6c` → **l** | `0x61` → **a** |
| x=51 | `#677b33` | `0x67` → **g** | `0x7b` → **{** | `0x33` → **3** |
| x=52 | `#615379` | `0x61` → **a** | `0x53` → **S** | `0x79` → **y** |
| x=53 | `#5f7231` | `0x5f` → **_** | `0x72` → **r** | `0x31` → **1** |
| x=54 | `#676854` | `0x67` → **g** | `0x68` → **h** | `0x54` → **T** |
| x=55 | `#3f7d96` | `0x3f` → **?** | `0x7d` → **}** | `0x96` → (非 ASCII) |

**最後一個像素的 B 通道為 `0x96` = 150**：這就是背景色 `#000096` 的殘留。因為 Flag 已經結束在 `}`，所以出題者把最後一個 byte 保留為背景值，讓顏色跟背景更接近，增加隱蔽性。

### 最終結果

依序拼接：**F l a g { 3 a S y _ r 1 g h T ? }**

```
Flag{3aSy_r1ghT?}
```

「3aSy_r1ghT?」是 leetspeak 風格的 "easy right?"，呼應題目的嘲諷語氣。

## 五、解題腳本

### 方法一：Python 腳本（處理 .htm 檔案）

```python
import re

with open("f.htm", "r") as f:
    content = f.read()

# 提取所有 y=2 那行的 box-shadow 像素
pattern = r"#([0-9a-fA-F]{6})\s+\d+px\s+2px"
matches = re.findall(pattern, content)

# 過濾異常色碼，轉換為 ASCII
flag = ""
for color in matches:
    if color != "000096":
        for i in range(0, 6, 2):
            v = int(color[i:i+2], 16)
            if 32 <= v <= 126:  # 可印字元範圍
                flag += chr(v)

print(f"Flag: {flag}")
# 輸出: Flag{3aSy_r1ghT?}
```

### 方法二：瀏覽器 Console（直接在頁面上執行）

在 `f.htm` 頁面（顯示黑色方塊那個）按 `F12`，切到「主控台」分頁，貼上：

```javascript
let style = document.querySelector('div').getAttribute('style');
let re = /#([0-9a-fA-F]{6})\s+\d+px\s+2px/g;
let m, flag = '';
while (m = re.exec(style)) {
  if (m[1] !== '000096') {
    [0,2,4].forEach(i => {
      let v = parseInt(m[1].substr(i,2), 16);
      if (v >= 32 && v <= 126) flag += String.fromCharCode(v);
    });
  }
}
console.log('Flag:', flag);
```

> **⚠️ 注意**：必須用 `getAttribute('style')` 取得原始的 hex 色碼。如果用 `div.style.boxShadow`，瀏覽器會自動把 `#000096` 轉成 `rgb(0, 0, 150)` 格式，導致比對失敗。這是實際解題時踩過的坑。

## 六、踩過的坑與教訓

### 坑 1：誤判題目類型

| 錯誤 | 正確 |
|---|---|
| 以為「輸入您的答案」是 Image Downloader 工具 | 它只是 CTF 的 Flag 提交欄 |
| 從 SSRF 角度思考繞過過濾器 | 應該先去看 Link 1 的圖片列表 |

**教訓**：先讀懂題目介面的每個元素，不要過度腦補。

### 坑 2：瀏覽器 Console 色碼格式轉換

| 錯誤 | 正確 |
|---|---|
| 用 `div.style.boxShadow` 取值 | 用 `div.getAttribute('style')` 取原始字串 |
| 用 `startsWith('#')` 比對色碼 | 用正則從原始字串提取 hex 值 |

**為什麼會錯？** 瀏覽器會對 CSS 屬性做標準化處理。`style.boxShadow` 回傳的是瀏覽器解析後的值（`rgb()` 格式），不是原始寫在 HTML 裡的 hex 格式。所以用 `#` 開頭去比對就什麼都抓不到，flag 輸出為空字串。

**教訓**：操作 DOM 時要注意瀏覽器會對 CSS 屬性做格式轉換，原始值和解析後的值可能不同。

### 坑 3：在錯誤的頁面執行腳本

| 錯誤 | 正確 |
|---|---|
| 在 `view-source:` 頁面執行 JS | 切到實際渲染的 `f.htm` 頁面 |
| `querySelector` 回傳 null，flag 為空 | 在正確頁面上才有 DOM 可以操作 |

**為什麼會錯？** `view-source:` 頁面是純文字顯示，沒有實際的 DOM 結構。`document.querySelector('div')` 在這個頁面找不到任何元素。

**教訓**：`view-source:` 頁面是純文字模式，沒有 DOM。要操作 DOM 必須在實際渲染的頁面上執行。

## 七、技術重點總結

### 核心技術：CSS Pixel Art 隱寫

| 項目 | 說明 |
|---|---|
| 渲染機制 | 用 `box-shadow` 逐像素畫圖，繞過傳統圖片格式 |
| 編碼方式 | 將 ASCII 字元編碼在 RGB 三個通道中 |
| 視覺隱蔽 | 異常色與背景色都是深色，肉眼無法分辨 |
| 下載保護 | 不是 `<img>` 標籤，無法右鍵另存 |
| 解題關鍵 | 檢查 HTML 原始碼 → 找異常像素 → RGB 轉 ASCII |

### 解題流程圖

```
無法右鍵另存
    ↓
副檔名是 .htm → 不是真正的圖片
    ↓
Ctrl+U 檢查原始碼 → 發現 box-shadow CSS Pixel Art
    ↓
背景是 #000096，用腳本找出異常色碼
    ↓
異常像素只有 6 個，資料量少 → 先試 ASCII 解碼
    ↓
第一個 byte 0x46 = 'F' → 匹配 Flag 格式，確認解法正確
    ↓
Flag{3aSy_r1ghT?}
```

### 參考知識

- **CSS box-shadow**：原本用於元素陰影效果，但可以用多個 box-shadow 逐像素畫圖
- **ASCII 表**：可印字元範圍 32~126，其中大寫字母 65~90，小寫 97~122，數字 48~57
- **Hex 轉換**：`#RRGGBB` 格式中，每兩位十六進位 = 1 個 byte (0~255)
- **Leetspeak**：用數字替代相似的字母，如 3=e、1=i、大小寫混用
